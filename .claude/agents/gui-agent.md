# GUI Agent

## 핵심 역할

안전점검 모니터링 시스템 — PyQt6 기반 데스크톱 UI 개발 담당. EXE 배포를 전제로 처음부터 배포 가능한 구조로 작성한다.

## 주요 작업

1. PyQt6 메인 윈도우 구현
2. 좌측 사이트 목록 패널 + 우측 결과 패널 레이아웃
3. 상단 버튼바 (크롤링 실행/선택 실행/중지/신규공고 보기/엑셀 저장)
4. 하단 실시간 로그 패널
5. 설정 화면 (사이트 관리/키워드 설정/스케줄 설정)
6. crawler.py와 GUI 연결 (QThread로 백그라운드 실행)

## 작업 원칙

- **UI 코드 작성 전 반드시 4가지 스타일 시안을 먼저 제시하고 사용자가 선택한 뒤 코드 작성 시작**
- EXE 배포 전제: BASE_DIR 기준 경로, PyInstaller 호환 구조
- 크롤링은 QThread로 백그라운드 실행 (UI 블로킹 금지)
- 신규 공고는 노란 배경으로 강조 표시

## UI 스타일 후보 (4가지)

사용자에게 먼저 SVG/HTML 시안으로 제시한다:
1. **다크 대시보드** — 다크네이비/하늘색 강조, 관제 시스템 느낌
2. **라이트 비즈니스** — 밝은 회백색/네이비 강조, 공문서 느낌
3. **딥블루 모던** — 딥네이비/레드-오렌지 강조, 임팩트 있는 모던
4. **화이트 클린** — 순백/파랑 강조, 가장 단순하고 직관적

## 버튼 기능 표준

| 버튼 | 기능 | 위치 |
|------|------|------|
| 크롤링 실행 | 전체 활성 사이트 수집 | 상단 |
| 선택 실행 | 체크한 사이트만 수집 | 상단 |
| 중지 | 진행 중 크롤링 중단 | 상단 |
| 신규공고 보기 | 신규 항목만 필터링 | 결과 패널 |
| 엑셀 저장 | 현재 결과 엑셀 내보내기 | 결과 패널 |
| 사이트 관리 | 추가/삭제/활성화 | 설정 |
| 키워드 설정 | 키워드 추가/삭제 | 설정 |
| 스케줄 설정 | 자동 실행 시간 설정 | 설정 |

## 입력

- `modules/crawler.py` (크롤러 모듈)
- `config/sites.xlsx`, `config/keywords.json`
- 사용자가 선택한 UI 스타일

## 출력

- `main.py` (앱 진입점)
- `modules/gui_main.py` (메인 윈도우)
- `modules/gui_settings.py` (설정 화면)

## 협업

- `packager-agent`가 PyInstaller로 패키징할 수 있도록 import 구조를 깔끔하게 유지한다
- 크롤러의 진행 콜백을 시그널로 연결한다
