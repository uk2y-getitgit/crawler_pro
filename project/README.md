## 1. 파일 구조

```
project/
├── main.py                     ← GUI 실행 진입점
├── main_crawl.py               ← CLI 수집 (GUI 없이 실행)
├── manual_board_editor.py      ← CLI 게시판 수동 편집기
├── .env                        ← API 키 및 설정 (Git 제외 필수)
├── requirements.txt
├── build.bat                   ← EXE 빌드 스크립트
│
├── config/
│   ├── sites.xlsx              ← 기관/게시판 목록 (핵심 설정 파일)
│   └── keywords.json           ← 키워드 필터 목록
│
├── data/
│   ├── board_finder_result.json  ← 자동 탐색 성공 결과
│   ├── board_finder_failed.json  ← 자동 탐색 실패 목록 (수동 등록 필요)
│   └── history.json            ← 수집 이력 (중복 방지용)
│
├── results/
│   └── 결과_YYYYMMDD.xlsx      ← 수집 결과 엑셀 (자동 생성)
│
└── modules/
    ├── gui_main.py             ← 메인 윈도우
    ├── gui_dialogs.py          ← 설정/게시판관리 다이얼로그
    ├── crawler.py              ← 수집 엔진
    ├── board_finder.py         ← 게시판 자동 탐색
    ├── ai_agent.py             ← AI 판단 (관련 공고 필터링)
    ├── reporter.py             ← 엑셀 출력
    └── scheduler.py            ← 자동 스케줄러
```

> **`config/sites.xlsx`** 가 모든 수집의 기준입니다. 이 파일에 게시판 URL이 등록되어야 수집이 작동합니다.

## 3. 환경 설정 (.env)

프로젝트 루트의 `.env` 파일을 열어 설정합니다.

```
ANTHROPIC_API_KEY=sk-ant-api03-...    ← Claude API 키
GEMINI_API_KEY=AIzaSy...              ← Gemini API 키
AI_PROVIDER=gemini                    ← 사용할 AI: gemini 또는 anthropic
CRAWL_DELAY=2                         ← 사이트 간 대기 시간(초)
MAX_PAGES=3                           ← 페이지당 최대 수집 페이지 수
LOG_LEVEL=INFO
```

> GUI의 **환경설정** 버튼에서도 변경 가능합니다.

**주의:** `.env` 파일은 절대 Git에 올리지 마세요. API 키가 노출됩니다.

## 6. 게시판 URL 수정 방법

> URL 수정은 수집 전에 반드시 완료해야 합니다. 빨간색(미등록) 기관은 수집이 불가합니다.

### 방법 1: GUI 게시판 관리 (권장)

1. 상단 툴바에서 **게시판 관리** 버튼 클릭
2. 왼쪽 기관 목록에서 수정할 기관 클릭
   - 빨간 배경 = 미등록, 우선 처리 필요
3. 오른쪽에서 게시판 추가/수정/삭제

#### 게시판 추가

1. 기관 선택 후 **+ 게시판 추가** 클릭
2. 다이얼로그에서 입력:
   - **게시판명**: 예) `고시공고`, `입찰공고`
   - **URL**: 게시판 목록 페이지 URL (예: `https://www.xxx.go.kr/notice/list.do`)
   - **페이지파라미터**: 페이지 이동 파라미터 (예: `?page=`, `&pageNo=`)
     - 자동 감지 가능한 경우 비워도 됩니다
3. **확인** 클릭 후 **저장** 버튼으로 최종 저장

#### 페이지파라미터 입력 방법

| 사이트 패턴 | 파라미터 입력 예시 |
|------------|-----------------|
| `list.do?page=2` | `?page=` |
| `list.do?pageNum=2` | `?pageNum=` |
| `list.do?currPage=2` | `?currPage=` |
| `list.do?p=2` | `?p=` |
| 자동 감지 가능 | (빈 칸) |

#### 게시판 수정

1. 기관 선택 → 오른쪽 테이블에서 수정할 게시판 행 클릭
2. **수정** 버튼 클릭 → 다이얼로그에서 변경
3. **저장** 클릭

#### 게시판 삭제

1. 행 선택 후 **삭제** 버튼 클릭 (또는 `Delete` 키)
2. **저장** 클릭

---

### 방법 2: 엑셀 일괄 가져오기

여러 기관을 한 번에 등록할 때 사용합니다.

1. **게시판 관리** → **입력 템플릿 다운로드** 클릭
2. 다운로드된 엑셀 파일에 데이터 입력:

| 기관명 | 게시판명 | URL | 페이지파라미터 |
|--------|---------|-----|--------------|
| 익산지방국토관리청 | 고시공고 | https://www.익산.go.kr/notice/list.do | ?page= |
| 세종시 | 입찰공고 | https://www.세종시.go.kr/bid/list.do | |

3. **엑셀로 일괄가져오기** 클릭 → 파일 선택
4. 가져오기 완료 후 **저장** 클릭

---

### 방법 3: sites.xlsx 직접 편집

`config/sites.xlsx` 를 Excel에서 직접 열어 편집합니다.

| 열 | 설명 |
|----|------|
| A: 기관명 | 기관 이름 (정확히 일치해야 함) |
| B: 사이트타입 | `일반` (기본값) |
| C: 게시판명 | 게시판 이름 |
| D: 게시판URL | 게시판 목록 페이지 전체 URL |
| E: 페이지파라미터 | 페이지네이션 파라미터 |
| F: 활성화 | `Y` = 수집, `N` = 제외 |
| G: 비고 | 메모 |

> **활성화** 컬럼이 `Y` 인 행만 수집됩니다. 제외할 기관은 `N` 으로 변경하세요.

## 7. 수집 실행 및 테스트

### 특정 기관만 테스트 수집

URL 등록 후 해당 기관만 먼저 테스트합니다.

1. 왼쪽 기관 목록에서 **전체 해제** 클릭
2. 테스트할 기관의 체크박스 선택
3. **선택 수집** 클릭
4. 로그 영역에서 수집 진행 상황 확인

### 전체 수집

1. **전체 수집** 클릭
2. 수집 중단이 필요하면 **수집 중지** 클릭 (현재 사이트 완료 후 중단)

### 수집 로그 확인 포인트

```
[OK] 대전광역시 / 고시공고 → 3건 수집
[OK] 논산시 / 안전점검공고 → 1건 수집 (신규 1건)
[SKIP] 익산지방국토관리청 → 게시판 미등록
[ERROR] 세종시 → 연결 실패 (ConnectionError)
```

- `[OK]` — 정상 수집
- `[SKIP]` — 게시판 URL 미등록 → 게시판 관리에서 URL 등록 필요
- `[ERROR]` — 사이트 접속 오류 → URL 확인 또는 사이트 점검 필요

### 수집 결과 필터링

수집 완료 후 상단 필터 바에서:

| 필터 | 사용법 |
|------|-------|
| 기관명 | 기관명으로 검색 |
| 제목 | 공고 제목 키워드 검색 |
| 게시판 | 게시판명 선택 |
| 신규만 | 오늘 새로 수집된 공고만 표시 |

## 8. 수집 결과 확인

수집이 완료되면 `results/결과_YYYYMMDD.xlsx` 파일이 자동 생성됩니다.

### 엑셀 시트 구성

| 시트명 | 내용 |
|--------|------|
| 신규공고 | 이번 수집에서 처음 발견된 공고 |
| 전체공고 | 수집된 모든 공고 (이력 포함) |
| 오류로그 | 수집 실패 기관 목록 및 오류 내용 |

### 컬럼 설명

| 컬럼 | 설명 |
|------|------|
| 신규여부 | ★ = 신규, 공백 = 기존 |
| 수집일시 | 수집된 날짜/시간 |
| 기관명 | 공고 기관 |
| 게시판명 | 수집된 게시판 |
| 게시글제목 | 공고 제목 |
| 게시일 | 공고 원문의 게시 날짜 |
| URL | 공고 원문 링크 |
| AI판단근거 | AI가 관련 공고로 판단한 이유 |

## 9. 자동 스케줄 설정

매일 지정 시각에 자동으로 수집하도록 설정합니다.

1. 툴바 **환경설정** 클릭
2. **자동 실행 스케줄** 섹션에서:
   - 체크박스 활성화
   - 실행 시각 설정 (예: `09:00`)
3. **저장** 클릭

프로그램이 실행 중인 동안 지정 시각에 자동으로 수집을 시작합니다.

## 11. 문제 해결

### 수집이 안 되는 경우

1. **게시판 미등록**: 기관 목록에서 빨간색 기관 → 게시판 관리에서 URL 등록
2. **URL 오류**: 브라우저에서 직접 URL을 열어 접속 확인
3. **페이지파라미터 오류**: 페이지 2로 이동할 때의 URL 파라미터 확인
   - 브라우저에서 게시판 2페이지로 이동 후 주소창 URL 확인

### AI 필터링이 너무 엄격한 경우

`.env` 에서 `AI_PROVIDER` 를 변경하거나, `config/keywords.json` 에 키워드를 추가합니다.

### 프로그램이 실행되지 않는 경우

```bash
python check_env.py
```

를 실행하여 환경 및 패키지 설치 상태를 확인합니다.

### 로그 파일 위치

- `gui_out.log` — 표준 출력 로그
- `gui_err.log` — 오류 로그
